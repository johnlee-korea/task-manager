# 업무 일정 관리 웹앱 (Task Manager)

## 프로젝트 개요

개인 업무 일정을 관리하는 웹 애플리케이션

## 핵심 기능

- 일정 등록/수정/삭제 (CRUD)
- 캘린더 뷰 (월간/주간/일간)
- 우선순위 표시 (높음/중간/낮음)
- 알림/리마인더 기능

## 작업 규칙

모든 작업은 반드시 아래 4단계를 순서대로 수행한다. 작업 규모에 따라 각 단계의 깊이는 유동적으로 조절한다 (큰 작업은 상세하게, 작은 작업은 간소화).

### 작업 시작
- 매 작업 시작 시 **어떤 작업인지 한 줄 요약**을 먼저 출력한다.

### 4단계 프로세스

1. **기획 단계**: 기능 목록과 구조를 정리하고, 승인 후 개발 시작. 작은 작업은 변경 범위와 방향만 간단히 정리.
2. **개발 단계**: 코드에 한국어 주석을 달고, 파일별 역할을 명확히 분리. 작은 작업이라도 코드 품질은 유지.
3. **검수 단계**: 버그 체크와 개선점을 리포트. 작은 작업은 핵심 동작 확인 수준으로 간소화.
4. **문서화 단계**: README 등 관련 문서를 최신 상태로 반영. 작은 작업은 변경 사항 요약 수준으로 간소화.

## 기술 스택

- HTML, CSS, JavaScript (순수 웹 기술로 시작)
- 추후 필요시 React 전환 가능
